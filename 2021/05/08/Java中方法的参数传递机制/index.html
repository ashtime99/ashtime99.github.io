Java中方法的参数传递机制

发表于 2021-05-08 分类于 java 阅读次数：
本文字数： 1.8k 阅读时长 ≈ 2 分钟

参数传递机制有两种：值传递和引用传递。

值传递：是指在调用函数时将实际参数复制一份传递到函数中，这样在函数中如果对参数进行修改，将不会影响到实际参数。

引用传递：是指在调用函数时将实际参数的地址直接传递到函数中，那么在函数中对参数所进行的修改，将影响到实际参数。

问题：Java中方法的参数传递机制是值传递还是引用传递？

java中无论参数是基本数据类型还是引用数据类型，都是值传递

例子：基本数据类型

很明显，方法中形参的修改并不会修改实际参数的值。在栈中，实参通过拷贝给形参值。

例子：引用数据类型

明明是说是值传递，怎么被修改了？

引用数据类型在new一个对象时，会在堆中开辟一个空间，栈通过地址来访问堆中空间。形参其实还是拷贝了实参在栈中的地址，因为地址相同，传递到的其实时同一个对象。对象的值虽然改变了，但是在栈中的地址并没有改变。

当我们把toString方法去掉，你就会发现地址并没有改变。所以，引用数据类型，也是值传递。
